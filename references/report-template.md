# 리포트 템플릿

핑퐁 완료 후 자동 생성하는 md 리포트의 표준 구조.
deliverable-engine 연동: 프레이밍(수신자+목적+핵심메시지) → 작성(모래시계) → QC.

---

## 리포트 표준 목차

```
1. Executive Summary
2. 현황 분석
3. 전환 목적 및 기대효과
4. 구조 설계 대안
5. 추천안 및 근거
6. 세무·법무 검토
7. 리스크 분석 및 대응 방안
8. 실행 로드맵
9. 재무 영향 분석
10. 별첨
```

---

## 각 챕터 상세

### 1. Executive Summary (1페이지)
- 핵심 결론 3문장
- 추천 구조 1줄 요약
- 기대효과 상위 3개
- 주요 리스크 상위 2개

### 2. 현황 분석
핑퐁에서 수집한 정보를 구조화.

| 섹션 | 내용 |
|------|------|
| 법인 구조 현황 | 법인 수, 지배관계도, 순환출자 여부 |
| 지분 현황 | 주주 구성, 오너 지분율, 특수관계자 |
| 재무 현황 | 자산규모, 부채비율, 매출 구성 |
| 사업 현황 | 사업 영역, 핵심/비핵심 구분 |
| 규제 적용 | 5천억 기준, 공정거래법 적용 여부 |

### 3. 전환 목적 및 기대효과
- 주목적 1개 + 부목적 1~2개 명시
- 정량적 기대효과(세금 절감액, 관리비용 절감 등) 가능한 범위 산출
- 정성적 기대효과(경영 유연성, 리스크 분리 등) 서술

### 4. 구조 설계 대안 (2~3개 시나리오)

각 시나리오별:
```
[시나리오 A: OOO형]

1) ASCII 구조도 (필수)
   예시:
   ┌─────────────┐
   │  OO홀딩스   │ ← 지주회사
   └──┬───┬───┬──┘
      │   │   │
   ┌──┴┐ ┌┴──┐ ┌┴──┐
   │A사│ │B사│ │C사│ ← 자회사
   └───┘ └───┘ └───┘

2) 전환 방식 + 전환 전후 비교
3) 장점 / 단점 (정량+정성)
4) 세무 영향 (지분율별 익금불산입 시뮬)
5) 규제 적합성 (5천억 기준 적용/비적용)
6) 실행 난이도 (상/중/하) + 예상 소요기간
```

**시나리오 비교 총괄표 (필수):**

| 항목 | 시나리오 A | 시나리오 B | 시나리오 C |
|------|-----------|-----------|-----------|
| 구조 유형 | | | |
| 전환 방식 | | | |
| 세금 절감(추정) | | | |
| 규제 리스크 | | | |
| 실행 난이도 | | | |
| 소요 기간 | | | |
| **추천 여부** | | | |

### 5. 추천안 및 근거
- 추천 시나리오 1개 확정
- 선정 근거 (왜 이것이 최적인지)
- 차선안과의 비교
- 전제 조건 명시

### 6. 세무·법무 검토

| 항목 | 현황 | 영향 | 대응 |
|------|------|------|------|
| 배당세(익금불산입) | 현재 지분율 → 불산입률 | 연간 세금 차이 | 지분 조정 전략 |
| 적격분할 | 요건 충족 여부 | 과세이연 규모 | 사후관리 계획 |
| 가업승계 | 적용 가능 여부 | 절세 규모 | 5년 관리 계획 |
| 지방세 | 입지 영향 | 중과세 규모 | 입지 전략 |

### 7. 리스크 분석 및 대응

| 리스크 | 발생확률 | 영향도 | 대응 방안 |
|--------|---------|--------|----------|
| (핑퐁에서 도출) | 상/중/하 | 상/중/하 | 구체적 대응 |

상위 3개 리스크에 대해 상세 시나리오 분석.

### 8. 실행 로드맵

```
Phase 1 (M0~M3): 준비 — 이사회 결의, 분할계획서, 주주총회
Phase 2 (M3~M6): 실행 — 분할기일, 공정위 신고, 등기
Phase 3 (M6~M12): 안정화 — 재상장, 사후관리 체계 구축
Phase 4 (M12~): 운영 — 모니터링, 최적화
```

마일스톤 테이블 + 담당(지주사/자문사/관청) 명시.

### 9. 재무 영향 분석

| 항목 | 전환 전 | 전환 후(추정) | 차이 |
|------|---------|-------------|------|
| 그룹 법인세 | | | |
| 배당 파이프라인 수익 | | | |
| 관리 오버헤드 | | | |
| NAV 할인율(상장 시) | | | |

### 10. 별첨
- 관련 법령 참조
- 벤치마크 사례 카드 (해당 사례만 선별)
- 상세 세무 시뮬레이션
- 용어 정의

## 리포트 QC 체크리스트

**1차 QC — 구조 검증:**

| # | 검사 | 기준 | FAIL 대응 |
|---|------|------|-----------|
| 1 | 구조도 존재 | §4 시나리오마다 ASCII 구조도 있음 | 구조도 추가 |
| 2 | 비교표 존재 | 시나리오 비교 총괄표 있음 | 비교표 추가 |
| 3 | 추천안 명시 | §5에 추천안 1개 + 근거 + 전제조건 | 추천안 확정 |
| 4 | 로드맵 타임라인 | §8에 Phase별 기간 + 담당 명시 | 타임라인 보완 |

**2차 QC — 정합성 검증:**

| # | 검사 | 기준 | FAIL 대응 |
|---|------|------|-----------|
| 5 | 숫자 정합 | 지분율·세율·금액이 축간 일치 | 불일치 수정 |
| 6 | 논리 완결 | 추천안 근거↔현황분석 정합 | 근거 보강 |
| 7 | 구조도↔본문 정합 | 구조도의 법인명·지분율이 본문과 일치 | 동기화 |
| 8 | 소스 위계 | ①공식>②전문>③일반. 단독 3차 ✗ | 소스 보강 |
| 9 | 세법 연도 | 모든 세무 수치에 기준연도 명시 | 연도 추가 |

**3차 QC — 표현 검증:**

| # | 검사 | 기준 | FAIL 대응 |
|---|------|------|-----------|
| 10 | 문체 | VOCAB 치환 완료 (MECE→삭제, spine→핵심메시지정합성) | 치환 |
| 11 | 확인필요 태그 | 불확실 정보에 태그 부착 | 태그 추가 |
| 12 | 전제조건 명시 | 추천안의 전제조건 §5에 명시 | 전제조건 추가 |
