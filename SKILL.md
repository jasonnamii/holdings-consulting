## 3단계 실행 흐름

```
①가이드 → ②핑퐁 → ③리포트
```

### ① 가이드 제시

**가이드 조건부:** 형이 해당 분야 경험자(이전 대화에서 확인 또는 명시)이면 ①가이드 스킵 → ②핑퐁 직행. 미확인 시에만 가이드 제시. 7축 전체 설명 → 시나리오 라우팅 후 필수축 2~3개만 설명으로 축소.

스킬 발동 시 클라이언트에게 전체 진행 가이드를 먼저 제공:

```
"홀딩스 컴퍼니 컨설팅을 시작합니다.

[진행 순서]
1. 현황 파악 — 몇 가지 질문을 드립니다 (약 10~15문항)
2. 구조 진단 — 7축 분석을 수행합니다
3. 리포트 — 맞춤 컨설팅 보고서를 생성합니다

준비되셨으면 시작하겠습니다."
```

가이드 제시 후 즉시 Phase 1 질문 시작. 승인 대기 불필요.

## §1. 7축 지식 체계

컨설팅의 지식 기반. 필요한 축만 선택 로딩.

| 축 | 핵심 | 스포크 |
|----|------|--------|
| 1. 법률·규제 | 공정거래법 요건, 설립방식 5종, 행위제한, 금융지주법 | → `references/knowledge-legal-tax.md §1` |
| 2. 세무·조세 | 익금불산입 4단계, 연결납세, 가업승계, 적격분할 | → `references/knowledge-legal-tax.md §2` |
| 3. 재무·자본구조 | 지분설계, 4대 수익원, NAV 디스카운트 | → `references/knowledge-finance-governance.md §1` |
| 4. 거버넌스 | 이사회, 상법398조, CP, 소수주주 보호 | → `references/knowledge-finance-governance.md §2` |
| 5. 전략·운영 | 전환목적 5유형, 시너지, 전환타이밍 | → `references/knowledge-strategy-cases.md §1` |

## §2. 시나리오 라우팅

핑퐁 초기 답변으로 시나리오를 판별하고, 이후 흐름을 분기.

| 시나리오 | 판별 조건 | 중점 축 | 핵심 산출물 |
|----------|----------|---------|------------|
| **신규설립** | 기존 법인 없음 or 백지 설계 | 1→2→3→5 | 구조설계안 + 세무시뮬 |
| **기존전환** | 복수법인 존재, 지주전환 검토 | 1→2→4→6 | 전환 로드맵 + 리스크분석 |
| **운영최적화** | 이미 지주체제, 효율 개선 | 3→4→5→6 | 포트폴리오 리밸런싱 + 거버넌스 개선안 |
| **승계설계** | 세대교체 목적 명시 | 2→1→4→3 | 승계 로드맵 + 세제최적화안 |
| **해외구조** | 해외법인 비중 큼 | 1→2(해외)→3 | 해외홀딩스 구조안 + CFC분석 |

**복합 시나리오 처리:**
- 주시나리오 1개 확정 + 부시나리오 최대 1개 보조 로딩
- 흔한 복합 패턴: 승계+전환(가장 빈번), 전환+해외, 최적화+승계
- 복합 시 중점 축 = 주시나리오 축 + 부시나리오에서 주시나리오에 없는 축만 추가

## §4. 10대 판단분기점

핑퐁 대화의 라우팅 로직. 각 분기점에서 답변이 달라지면 컨설팅 방향이 전면 변경됨.

| # | 분기점 | 영향 |
|---|--------|------|
| 1 | 자산 5,000억 이상/이하 | 공정거래법 적용 여부 → 규제 강도 결정 |
| 2 | 인적분할 vs 물적분할 | 지분구조·세무·소수주주 이슈 전면 변경 |
| 3 | 지분율 경계값(50/30/20/10%) | 배당세 급변 → 지분 설계 방향 |
| 4 | 금융업 포함 여부 | 금융지주회사법 별도 적용 |
| 5 | 승계 목적 유무 | 가업승계 특례·증여세 전략 활성화 |
| 6 | 상장 vs 비상장 | 의무 지분율 차이(30% vs 50%) |
| 7 | 해외 자회사 유무 | CFC·이전가격·조세조약 검토 |
| 8 | 순환출자 존재 여부 | 해소 의무, 전환 복잡도 |
| 9 | 단층 vs 다층(중간지주) | 규제 준수 방식, 경영 유연성 |
| 10 | 전환 vs 신규설립 | 기존 법인 재편 vs 백지 설계 |

## §5. 품질 게이트

| 단계 | 검증 | FAIL 시 |
|------|------|---------|
| 핑퐁 완료 | 10대 분기점 중 해당 항목 모두 답변 확보 | 미확보 분기점 목록 제시 → 추가 질문 1회 |

## TURBO 모드 — 고품질 병렬 가속화

**목적:** 홀딩스 컨설팅 리포트 시간 단축. 10대 판단분기점·시나리오 라우팅·품질 게이트·법률/세무 정확성 불변, 재무·거버넌스·전략·사례 축과 리포트 섹션 드래프트만 Agent 분산.

**트리거:** "터보로 진단" · "TURBO" · "홀딩스 터보" 명시. 미명시 = 일반.

**원칙 (4):**
1. **병렬화만, 스킵 금지** — 핑퐁·10대 판단분기점 적용·법률/세무 판단·§5 품질 게이트 불변
2. **독립성 확인** — 축 3·4·5·6은 상호 독립 분석 가능. 축 1·2는 기초 전제 제공 후에만
3. **통합 시퀀셜** — 시나리오 확정·추천안 판정·축간 교차 일관성은 메인
4. **Agent 브리프 완결성** — 핑퐁 답변·시나리오·10대 분기점 판정·해당 축 스포크 발췌·VOCAB 치환 규칙 전달

**병렬 타겟:**
- §1 축 3(재무)·4(거버넌스)·5(전략)·6(사례) 스포크 로드·분석 — 축별 Agent (법률·세무 판단 확정 후 실행)
- §3 리포트 섹션별 드래프트: 현황 진단·구조 대안 A~C·리스크 분석·로드맵 — 섹션별 Agent

**병렬 제외 (시퀀셜·중요):**
- §1 축 1(법률·규제)·축 2(세무·조세) — 오류 시 치명적. 지분율·세율·기준연도 직접 확인
- § 핑퐁 Phase 1~3 (형 답변 대기·추가 질문 분기)
- §2 시나리오 라우팅 (주/부 시나리오 확정)
- §4 10대 판단분기점 적용 (자산 5천억·분할방식·지분율 경계 등)
- §5 품질 게이트 (숫자정합·논리완결·문체·축간 교차·세법 기준연도·구조도↔본문 정합)
- 최종 추천안 확정·전제조건·"확인필요" 태그

**품질 저하 방어:**
- 축간 숫자 불일치(지분율·세율·금액) 감지 → 스포크 재로드·시퀀셜 재작성
- 내부용어("MECE"·"spine") 리포트 혼입 감지 → VOCAB 치환 재적용
- 법률·세무 기준연도 미명시·축 1·2 판단이 축 3~6에 전제로 안 잡힌 경우 → TURBO 중단 후 시퀀셜
- 추천안 없이 대안 나열·대기업 프레임을 중소에 적용 감지 → 재작성

**예상 단축:** 30~45% (축 수·리포트 섹션 수·시나리오 복합성에 비례. 법률/세무 비중 큰 케이스는 단축 폭 제한적)
